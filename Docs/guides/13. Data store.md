## Introducción
En el desarrollo móvil moderno, es fundamental contar con mecanismos eficientes para persistir datos. **Data Store** es la solución recomendada por Android para reemplazar a `SharedPreferences`, permitiendo el almacenamiento de datos de forma segura, asíncrona y eficiente.

Existen dos tipos:
- **Preferences Data Store:** Almacena pares clave-valor (similar a SharedPreferences).
- **Proto Data Store:** Almacena datos estructurados usando Protocol Buffers.

En esta guía nos centraremos en **Preferences Data Store**.

## SharedPreferences vs Data Store

| Característica | SharedPreferences | Data Store |
| --- | --- | --- |
| **Modelo** | Basado en XML | Basado en archivos binarios |
| **Tipo de datos** | Clave-valor simples | Clave-valor o estructurados |
| **Seguridad** | No cifrado por defecto | Soporta cifrado y es más seguro |
| **API** | Síncrona (bloqueante) | Asíncrona (Corrutinas/Flow) |
| **Manejo de errores** | Propenso a errores de concurrencia | Manejo robusto de errores |

## Implementación de Data Store

### 1. Agregar dependencias
En `libs.versions.toml`:
```toml
[versions]
dataStore = "1.2.0"

[libraries]
data-store = { module = "androidx.datastore:datastore-preferences", version.ref = "dataStore" }
```

En `build.gradle.kts`:
```kotlin
dependencies {
    implementation(libs.data.store)
}
```

### 2. Configurar el Singleton (`SessionDataStore.kt`)
Cree este archivo en `data/datastore`. Gestionará la persistencia de la sesión:

```kotlin
private val Context.dataStore: DataStore<Preferences> by preferencesDataStore(name = "session")

@Singleton
class SessionDataStore @Inject constructor(
    @ApplicationContext private val context: Context
) {
    private object Keys {
        val USER_ID = stringPreferencesKey("user_id")
        val ROLE = stringPreferencesKey("role")
    }

    val sessionFlow: Flow<UserSession?> = context.dataStore.data.map { prefs ->
        val userId = prefs[Keys.USER_ID]
        val roleStr = prefs[Keys.ROLE]
        if (userId.isNullOrBlank() || roleStr.isNullOrBlank()) null
        else UserSession(userId, UserRole.valueOf(roleStr))
    }

    suspend fun saveSession(userId: String, role: UserRole) {
        context.dataStore.edit { prefs ->
            prefs[Keys.USER_ID] = userId
            prefs[Keys.ROLE] = role.name
        }
    }

    suspend fun clearSession() {
        context.dataStore.edit { it.clear() }
    }
}
```

### 3. Modelo de Datos (`UserSession.kt`)
```kotlin
data class UserSession(val userId: String, val role: UserRole)
```

### 4. ViewModel de Sesión (`SessionViewModel.kt`)
```kotlin
sealed interface SessionState {
    data object Loading : SessionState
    data object NotAuthenticated : SessionState
    data class Authenticated(val session: UserSession) : SessionState
}

@HiltViewModel
class SessionViewModel @Inject constructor(
    private val sessionDataStore: SessionDataStore
) : ViewModel() {

    val sessionState: StateFlow<SessionState> = sessionDataStore.sessionFlow.map { session ->
        if (session != null) SessionState.Authenticated(session) 
        else SessionState.NotAuthenticated
    }.stateIn(
        scope = viewModelScope,
        started = SharingStarted.WhileSubscribed(5000),
        initialValue = SessionState.Loading
    )

    fun login(userId: String, role: UserRole) = viewModelScope.launch {
        sessionDataStore.saveSession(userId, role)
    }

    fun logout() = viewModelScope.launch {
        sessionDataStore.clearSession()
    }
}
```

## Integración en la UI (`AppNavigation.kt`)
Podemos usar el estado de la sesión para decidir qué flujo de navegación mostrar:

```kotlin
@Composable
fun AppNavigation(sessionViewModel: SessionViewModel = hiltViewModel()) {
    val sessionState by sessionViewModel.sessionState.collectAsState()

    Surface(modifier = Modifier.fillMaxSize()) {
        when (val state = sessionState) {
            is SessionState.Loading -> CircularProgressIndicator()
            is SessionState.NotAuthenticated -> AuthNavigation()
            is SessionState.Authenticated -> MainNavigation(
                session = state.session, 
                onLogout = sessionViewModel::logout
            )
        }
    }
}
```
