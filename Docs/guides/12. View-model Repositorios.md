## Introducción
En esta guía profundizaremos en el uso de **Repositorios** para gestionar la lógica de datos y mejorar la separación de responsabilidades. El objetivo es crear una aplicación escalable donde los ViewModels solo preparen datos para la UI, mientras que los repositorios manejen el acceso a datos (API, DB, etc.).

### Capas de la Arquitectura
1.  **Capa de Presentación:** Pantallas (UI) y ViewModels.
2.  **Capa de Dominio:** Interfaces de repositorios y modelos de datos.
3.  **Capa de Datos:** Implementaciones concretas de repositorios.
4.  **Capa de Inyección de Dependencias (DI):** Configuración de Hilt para proveer dependencias.

## Conceptos Clave

### ViewModel
Actúa como intermediario entre la vista y el modelo. Su responsabilidad es gestionar el estado de la UI de forma independiente al ciclo de vida de la actividad.

### Repositorios
Capa que abstrae el origen de los datos. Se divide en:
-   **Interfaz:** Define las operaciones (se ubica en la capa de Dominio).
-   **Implementación:** Lógica concreta de acceso a datos (se ubica en la capa de Datos).

### Hilt y KSP
-   **Hilt:** Biblioteca de Google para inyección de dependencias (DI) que automatiza la creación de instancias.
-   **KSP (Kotlin Symbol Processing):** Herramienta moderna para procesar anotaciones en Kotlin, más eficiente que Kapt.

## Integración en el Proyecto

### 1. Agregar dependencias
Actualice su archivo `libs.versions.toml`:

```toml
[versions]
hiltAndroid = "2.57.2"
hiltNavigationCompose = "1.3.0"
ksp = "2.1.21-2.0.1"

[libraries]
androidx-hilt-navigation-compose = { module = "androidx.hilt:hilt-navigation-compose", version.ref = "hiltNavigationCompose" }
hilt-android = { module = "com.google.dagger:hilt-android", version.ref = "hiltAndroid" }
hilt-compiler = { module = "com.google.dagger:hilt-compiler", version.ref = "hiltAndroid" }

[plugins]
hilt-android = { id="com.google.dagger.hilt.android", version.ref="hiltAndroid"}
devtools-ksp = { id="com.google.devtools.ksp", version.ref="ksp" }
```

En el `build.gradle` del proyecto:
```kotlin
plugins {
    alias(libs.plugins.hilt.android) apply false
    alias(libs.plugins.devtools.ksp) apply false
}
```

En el `build.gradle` del módulo:
```kotlin
plugins {
    alias(libs.plugins.hilt.android)
    alias(libs.plugins.devtools.ksp)
}

dependencies {
    implementation(libs.hilt.android)
    ksp(libs.hilt.compiler)
    implementation(libs.androidx.hilt.navigation.compose)
}
```

### 2. Configurar la Clase de Aplicación
Cree `MyApp.kt` y anótela con `@HiltAndroidApp`:
```kotlin
@HiltAndroidApp
class MyApp : Application()
```

Indíquela en el `AndroidManifest.xml`:
```xml
<application android:name=".MyApp" ...>
```

### 3. Configurar Activity Principal
Añada la anotación `@AndroidEntryPoint` en `MainActivity.kt`.

### 4. Definir el Repositorio
**Interfaz (`domain/repository/UserRepository.kt`):**
```kotlin
interface UserRepository {
    val users: StateFlow<List<User>>
    fun save(user: User)
    fun findById(id: String): User?
    fun login(email: String, password: String): User?
}
```

**Implementación (`data/repository/UserRepositoryImpl.kt`):**
```kotlin
@Singleton
class UserRepositoryImpl @Inject constructor() : UserRepository {
    private val _users = MutableStateFlow<List<User>>(emptyList())
    override val users = _users.asStateFlow()

    init { _users.value = fetchUsers() }

    override fun save(user: User) { _users.value += user }
    override fun findById(id: String) = _users.value.firstOrNull { it.id == id }
    override fun login(email: String, password: String) = 
        _users.value.firstOrNull { it.email == email && it.password == password }
}
```

### 5. Vincular con Hilt (`di/RepositoryModule.kt`)
```kotlin
@Module
@InstallIn(SingletonComponent::class)
abstract class RepositoryModule {
    @Binds
    @Singleton
    abstract fun bindUserRepository(impl: UserRepositoryImpl): UserRepository
}
```

## Uso en ViewModels y Pantallas
Inyecte el repositorio en el constructor del ViewModel:

```kotlin
@HiltViewModel
class UserListViewModel @Inject constructor(
    private val repository: UserRepository
) : ViewModel() {
    val users = repository.users
}
```

En el Composable, use `hiltViewModel()`:

```kotlin
@Composable
fun UserListScreen(
    userViewModel: UserListViewModel = hiltViewModel()
) { ... }
```
