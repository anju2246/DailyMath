## Introducción
Toda aplicación que gestione datos debe tener definidas sus entidades del dominio. Una entidad del dominio representa un objeto o concepto central en el sistema. 

Por ejemplo:
- En una app de tareas: **Tarea** (título, descripción, fecha, estado).
- En una app de e-commerce: **Producto** (nombre, precio, descripción, categoría).

Definir estas entidades es crucial, ya que forman la base sobre la cual se construye la lógica de negocio y las funcionalidades.

## Definición de Entidades del Dominio
Trabajaremos con una aplicación donde los ciudadanos reportan problemas urbanos y los administradores los gestionan.

### Reporte (`Report`)
- **id**: Identificador único.
- **title**: Título del reporte.
- **description**: Detalle del problema.
- **location**: Ubicación geográfica (`Location`).
- **status**: Estado (`ReportStatus`).
- **type**: Tipo de problema (hueco, alumbrado, basura, etc.).
- **photoUrl**: URL de la foto.
- **ownerId**: ID del usuario creador.

### Ubicación (`Location`)
- **latitude**: Latitud.
- **longitude**: Longitud.

### Usuario (`User`)
- **id**: Identificador único.
- **name**: Nombre completo.
- **email**: Correo electrónico.
- **phoneNumber**: Teléfono.
- **city**: Ciudad.
- **address**: Dirección (opcional).
- **role**: Rol (`UserRole`).
- **profilePictureUrl**: Foto de perfil.
- **password**: Contraseña segura.

## Creación de las Entidades del Dominio
Cree los siguientes archivos en el paquete `domain.model`:

### `Report.kt`
```kotlin
data class Report(
    val id: String,
    val title: String,
    val description: String,
    val location: Location,
    val status: ReportStatus,
    val type: String,
    val photoUrl: String,
    val ownerId: String
)
```

### `Location.kt`
```kotlin
data class Location(
    val latitude: Double,
    val longitude: Double
)
```

### `User.kt`
```kotlin
data class User(
    val id: String,
    val name: String,
    val city: String,
    val address: String,
    val email: String,
    val password: String,
    val phoneNumber: String = "",
    val profilePictureUrl: String = "",
    val role: UserRole = UserRole.USER
)
```

### `ReportStatus.kt`
```kotlin
enum class ReportStatus {
    PENDING, IN_PROGRESS, RESOLVED
}
```

### `UserRole.kt`
```kotlin
enum class UserRole {
    USER, ADMIN
}
```

> [!TIP]
> **Diferencia entre `class` y `data class`:** Una `data class` en Kotlin está diseñada para almacenar datos y proporciona automáticamente métodos como `equals()`, `hashCode()`, `toString()` y `copy()`.

## Múltiples Elementos en Compose
En Jetpack Compose, existen componentes "perezosos" (Lazy) que permiten mostrar listas largas de manera eficiente, renderizando solo lo que es visible.

### LazyColumn y LazyRow
Equivalentes al `RecyclerView` de las vistas tradicionales. Incluyen scroll automático.

```kotlin
LazyColumn {
    items(itemsList) { item ->
        Text(text = item.name)
    }
}
```

### Grid
Para organizar elementos en cuadrículas usamos `LazyVerticalGrid` o `LazyHorizontalGrid`.

```kotlin
LazyVerticalGrid(columns = GridCells.Fixed(2)) {
    items(itemsList) { item ->
        Text(text = item.name)
    }
}
```

## Uso en la Aplicación de Reportes Urbanos

### 1. Crear `UserListScreen`
Cree este composable en `features/users/list`:

```kotlin
package com.example.demoapp.features.users.list

import androidx.compose.foundation.clickable
import androidx.compose.foundation.layout.*
import androidx.compose.foundation.lazy.LazyColumn
import androidx.compose.foundation.lazy.items
import androidx.compose.material3.*
import androidx.compose.runtime.*
import androidx.compose.ui.Modifier
import androidx.compose.ui.draw.clip
import androidx.compose.ui.unit.dp
import androidx.lifecycle.viewmodel.compose.viewModel
import com.example.demoapp.domain.model.User
import coil3.compose.AsyncImage

@Composable
fun UserListScreen(
    onNavigateToUserDetail: (String) -> Unit,
    usersViewModel: UserListViewModel = viewModel()
) {
    val users by usersViewModel.users.collectAsState()

    LazyColumn(modifier = Modifier.fillMaxSize()) {
        items(users) { user ->
            ItemUser(user = user, onNavigateToUserDetail = onNavigateToUserDetail)
        }
    }
}

@Composable
fun ItemUser(user: User, onNavigateToUserDetail: (String) -> Unit) {
    ListItem(
        modifier = Modifier
            .clip(MaterialTheme.shapes.small)
            .clickable { onNavigateToUserDetail(user.id) },
        headlineContent = { Text(text = user.name) },
        supportingContent = { Text(text = user.email) },
        leadingContent = {
            AsyncImage(
                model = user.profilePictureUrl,
                contentDescription = "Imagen de perfil",
                modifier = Modifier.size(60.dp).clip(CircleShape)
            )
        }
    )
}
```

### 2. Crear `UserListViewModel`
```kotlin
package com.example.demoapp.features.users.list

import androidx.lifecycle.ViewModel
import com.example.demoapp.domain.model.*
import kotlinx.coroutines.flow.*

class UserListViewModel : ViewModel() {
    private val _users = MutableStateFlow<List<User>>(emptyList())
    val users: StateFlow<List<User>> = _users.asStateFlow()

    init {
        fetchUsers()
    }

    private fun fetchUsers() {
        _users.value = listOf(
            User(id = "1", name = "Juan", city = "Armenia", email = "juan@email.com", password = "123", profilePictureUrl = "https://picsum.photos/200"),
            User(id = "2", name = "Maria", city = "Pereira", email = "maria@email.com", password = "456", profilePictureUrl = "https://picsum.photos/201")
        )
    }
}
```
